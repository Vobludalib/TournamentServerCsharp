Detailed Description

See Tournament.TournamentLockHandler for explanation why these exist.

Note that for sets, I have opted to lock the entire set as a whole, and not individual fields - in most use cases, only one user will be sending requests to read/edit any given set, so it would be too much overhead to have such fine control. This can be changed if it is found that the aforementioned is actually a common usecase.
